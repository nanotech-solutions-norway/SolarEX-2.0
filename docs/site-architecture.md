# SolarEX Site Architecture

## Objective
Build a commercially credible SolarEX website that converts technical source material into clear buying pathways for solar asset owners, EPCs, O&M providers, and industrial stakeholders.

## Primary audiences
- Utility-scale solar owners and operators
- EPC contractors
- O&M providers
- Industrial asset managers
- Regional distribution or channel partners

## Information architecture

### 1. Home
Purpose: fast strategic positioning.
Sections:
- hero value proposition
- two-product overview
- proof metrics
- why SolarEX
- application snapshot
- ROI teaser
- CTA strip

### 2. Products
#### Quartz (SiO₂)
Position as passive hydrophobic / oleophobic self-cleaning solution.
Focus:
- easy-clean behavior
- dust, water, and contamination release
- durability
- EU and Middle East ROI narratives
- warm-climate formulation note

#### Titan (TiO₂)
Position as active photocatalytic self-cleaning solution.
Focus:
- UV-triggered photocatalysis
- superhydrophilic water sheeting
- 360-day field study framing
- organic contamination handling
- protection and long-duration performance

### 3. Technology
Purpose: explain how SiO₂ and TiO₂ work and when each is strategically relevant.
Modules:
- passive vs active self-cleaning
- hydrophobic vs superhydrophilic behavior
- nanostructure and coating thickness
- application and curing

### 4. ROI
Purpose: move from curiosity to decision support.
Modules:
- payback framing
- region-specific examples
- gain drivers
- maintenance reduction logic
- future interactive calculator placeholder

### 5. Proof
Purpose: strengthen commercial trust.
Modules:
- field trial summary
- study-based claims
- technical specifications
- validation framework
- future downloadable evidence library

### 6. Applications
Purpose: show where the product family fits.
Initial application buckets:
- utility-scale solar
- C&I rooftops
- high-dust geographies
- humid / organic contamination environments
- water-scarce maintenance environments

### 7. Contact
Purpose: conversion.
CTAs:
- request evaluation
- request technical discussion
- request commercial quotation
- request pilot discussion

## Navigation strategy
Top nav:
- Home
- Quartz
- Titan
- Technology
- ROI
- Proof
- Applications
- Contact

## Content principles
- avoid unsupported universal claims
- distinguish source-based results from general positioning
- keep technical language accessible but credible
- maintain product separation while allowing compare logic later
